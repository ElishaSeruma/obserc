Obserc Beta Release Notes

Every update brings us closer to launch. Below is a detailed breakdown of changes, improvements, and known issues for each Obserc beta release.

Obserc Beta v0.01 Release Notes

Release Date: August 27, 2025

Thank you for participating in the closed beta for Obserc! This initial version is focused on testing the core detection pipeline and local data logging. Your feedback on the stability and accuracy of these foundational features is crucial.

Key Features & Beta Goals

This release includes the essential framework for on-device analysis and data collection.

Emotional & Verbal Abuse Detection

The primary goal of this beta is to validate our manipulation models. The app actively listens for speech and classifies it as either "manipulative" or "non-manipulative."

Listening Modes

  • Triggered Sessions: Users can manually start and stop a listening session from the Home screen. This is the primary mode for controlled testing.
  • Background Listening (Toggle): A toggle is available in Settings to enable a persistent background listening service.

Local Data Logging

All session data and detected events are stored exclusively on your device in a local database (Room DB). No audio or personal data is sent to the cloud in this version.

Session History

A functional History screen displays a list of all completed sessions. Tapping a session reveals a detailed view with timestamps and detected phrases.

Predefined Test Circle

The Circle screen displays a static, read-only list of internal test members. The "Send Test Alert" button confirms backend connectivity by writing a timestamped event to our database.

Daily Check-in (Placeholder)

The UI for the daily check-in feature is included to gather feedback on the concept. The feature itself is planned for v0.02.

Settings Control

The Settings screen allows you to toggle background listening and other data collection preferences. These settings are saved locally on your device.

How to Test

  1. Run Triggered Sessions: Use the "Start Triggered Listening Session" button on the Home screen. Speak scripted or ad-libbed phrases to test detection accuracy.
  2. Review Session History: After stopping a session, navigate to the History tab to see if the session was logged correctly. Tap on it to view the details.
  3. Test the Database Connection: Go to the Circle tab and press the "Send Test Alert" button. Please let the development team know if you encounter any errors.
  4. Explore the UI: Please navigate through all screens (Home, History, Circle, Settings) and report any crashes, layout issues, or unexpected behavior.

Known Issues

  • Speech Recognition: The built-in Android Speech Recognition requires a stable internet connection and may not be available on all emulator configurations. A device with Google Play Services is required.
  • Performance: You may notice a slight delay or UI stutter when stopping a session or submitting a check-in as data is written to the local database.

What's Next in v0.02 (Planned for September 30th)

The next major beta version will build upon this foundation to create a fully integrated and dynamic user experience. We are excited about the significant advancements planned, which will dramatically expand the app's capabilities.

Full User Interface & Dynamic Login

We will move beyond the single-tester model to a complete UI with a dynamic, multi-faceted authentication system. This will allow for secure, individual user accounts, personalized profiles, and a tailored user experience.

Expanded Verbal Analysis

While v0.01 focuses specifically on patterns of mental and emotional abuse, v0.02 will broaden the analytical scope. The underlying models will be enhanced to detect a wider range of harmful speech, including patterns of coercion, direct threats, and other verbal indicators of escalating aggression.

Video Analysis for Physical Abuse Detection

A significant new feature will be the introduction of video analysis. With explicit user consent, the app will be able to process video streams to identify patterns of movement and interaction that may indicate physical abuse or escalating violence. This expands Obserc's focus from the purely verbal (v0.01) to include critical non-verbal cues, aiming to provide a more holistic safety net.

Full Circle Integration & Intelligent Escalation

The Circle feature will become fully dynamic. Users will be able to securely invite and manage their own trusted contacts. The escalation logic will be implemented in full, moving beyond the simple test button. This system will be designed to intelligently assess the severity and frequency of detected events—both verbal and physical—to trigger a tiered alert system, ensuring that Circle members are notified appropriately when a situation appears to be escalating.

Thank you again for your time and effort in testing Obserc. Your contribution is invaluable.

Obserc Beta v0.02 Release Notes

Release Date: September 30, 2025

Thank you for your invaluable feedback on v0.01! Your testing of the core pipeline has allowed us to take a major leap forward. Beta v0.02 transitions Obserc from a foundational proof-of-concept into a fully integrated and dynamic user experience. This release introduces user accounts, significantly expanded detection capabilities, and the full implementation of the 'Circle' alert system.

What's New in v0.02

This version brings to life the core features of the Obserc vision, moving beyond basic logging to a multi-faceted safety and alert system.

Full User Accounts & Dynamic Login

The single-tester model is gone. You can now create a secure, individual user account . This enables personalized profiles and a tailored user experience, forming the foundation for all future features.

Expanded Verbal Analysis: Beyond Manipulation

The detection model has been significantly upgraded. In addition to manipulative language, the app can now identify a wider range of harmful speech patterns, including:

  • Coercion
  • Direct Threats
  • Verbal indicators of escalating aggression

Introducing Video Analysis for Physical Abuse Detection

This is a major new feature that expands Obserc's focus beyond verbal cues. With explicit user consent, you can now enable video analysis during a session. The app will process the video stream locally to identify patterns of movement and interaction that may indicate physical abuse or escalating violence, providing a more holistic safety net.

Fully Dynamic 'Circle' & Intelligent Alert Escalation

The 'Circle' is no longer a static list. You can now securely invite and manage your own list of trusted contacts. The "Send Test Alert" button has been replaced with a fully implemented intelligent escalation system. This system assesses the severity and frequency of detected events—both verbal and physical—to trigger a tiered alert to your Circle members when a situation appears to be escalating.

How to Test

Your primary goal is to test the integration and functionality of these new, dynamic systems.

  1. Create Your Account: Log out of the test profile and go through the new user registration and login flow.
  2. Build Your Circle: Navigate to the Circle screen and try inviting a contact (you can use a secondary email address for testing). Confirm that you can add and remove members.
  3. Test Expanded Detection (Verbal & Video):
    • Run triggered sessions to test the new verbal classifiers. Use phrases that imply coercion or threats.
    • With your consent, start a session with video analysis enabled. Safely simulate scenarios with sudden movements or aggressive gestures to test the detection model's accuracy.
  4. Trigger a Test Escalation: In a controlled session, try to generate multiple detected events (both verbal and non-verbal) to test if the intelligent escalation logic triggers a notification as expected.

Known Issues

  • Performance & Battery: Enabling video analysis is processor-intensive and will have a noticeable impact on battery life and may cause the device to warm up during long sessions.
  • Model Accuracy: The new verbal and video analysis models are in their early stages. Please report any false positives (incorrectly flagged events) or false negatives (missed events) with as much detail as possible. This feedback is critical for refinement.
  • Network Dependency: Circle invitations and alert escalations require a stable internet connection to function correctly.

Thank you for continuing this journey with us. Your feedback on these powerful new features is essential as we work to build a more comprehensive and responsive safety tool.
